## 2. Rules to ADD to AGENTS.md

### 2A. Goal Creation Protocol (NEW SECTION — insert after Task Creation Protocol)

```markdown
## 📎 Goal Creation Protocol

When ANY goal is created or updated in ~/Master/goals/:

### Step 1: Copy Template
Copy `~/Master/task-system/goal-template-v1.3.md` → `~/Master/goals/GOAL-XXX.md`

### Step 2: Fill Header
Required: Name, One-liner, Metric (one number), Measurement source, Deadline, Status=ACTIVE, Mode=BUILDING.
Link: `Strategy Change Protocol: strategy-change-protocol.md`

### Step 3: Assess Foundations
For EACH of the 8 foundations (Offer, Audience, Channel, Conversion Mechanism, Delivery System, Revenue Model, Legal Framework, Cost Structure):
- Assess current state and maturity (raw → draft → tested → proven)
- Mark sufficient (✅) or insufficient (❌)
- If insufficient: identify or create the implementing task
- **Insufficient foundations = infrastructure blockers → priority unblock**

### Step 4: Define Strategy Structure
- Decompose into sub-goals (each with own metric)
- Define mechanism chain per sub-goal (conversion funnel with ratios)
- Identify the bottleneck (worst ratio or lowest throughput)
- Define paths (channels/tactics feeding mechanism steps)
- Designate ONE path as MVP (fewest requirements, fastest to revenue)

### Step 5: Set Gates and Stop Criteria
- Define threshold gates for deferred work (quantitative unlock conditions)
- Define system stop criteria (what must be true for BUILDING → OPERATIONAL)

### Step 6: Create Execution Tasks
- Create tasks for MVP path + insufficient foundations
- Each task MUST set `Goal:` field (see Task Creation Step 6b below)
```

```markdown
### Step 6b: Goal Linkage (MANDATORY)
- Set `Goal:` field to: `GOAL-ID | PATH-ID | MZ: StepName`
  - Example: `GOAL-001 | PATH-LI | M1: Outreach`
- If no goal exists for this task, one of:
  - Link to an existing goal (most tasks should)
  - Flag: "⚠️ ORPHAN — no goal linkage. Create goal or justify standalone."
  - Standalone tasks (one-off admin, meta-system work) may use `Goal: STANDALONE | [reason]`
- **4-Hop Trace Rule:** Every task must trace to a goal in ≤4 hops: Task → Path → MechanismStep → SubGoal → Goal. If the trace breaks, fix the linkage.

### Step 6c: Set Location
- Set `Location:` field: 🏠 Remote | 📍 On-site | 🔄 Either
- If on-site only, note what would be needed to make it remote (feeds automation planning)

### Step 6d: Decision Log
- If the task creation involved any non-obvious decision (priority override, unusual classification, dependency choice), log it in the task's Decision Log section.
- Format: `| Date | Decision | Rationale | Reversible? |`
```

### 2C. Strategy/Execution Boundary Rule (NEW SECTION)

```markdown
## 🚫 Strategy/Execution Boundary

**You cannot change strategy from execution mode.**

During daily task execution:
- If you notice something that suggests the strategy is wrong → capture it as a note in the goal file's Decision Log with prefix "📝 OBSERVATION:"
- Do NOT: change paths, redefine mechanism chains, add/drop sub-goals, or pivot approach
- Act on observations during the next **weekly strategy review** only

This applies to ALL agents — main session, sub-agents, cron jobs, Atlas, anyone.

### What counts as strategy vs execution:
| Strategy (weekly review only) | Execution (daily, do freely) |
|-------------------------------|------------------------------|
| Add/drop/change a path | Create/complete/update tasks |
| Change mechanism chain structure | Reorder task priority |
| Modify sub-goal metrics | Mark tasks done |
| Declare a foundation sufficient/insufficient | Update execution logs |
| Change goal mode (BUILDING↔OPERATIONAL) | Score and classify new tasks |
| Override bottleneck designation | Note observations |
```

## ☀️ Morning Briefing Protocol

Generate daily. Format:

## Context (auto-generated)
[For each active goal]: GOAL-ID [MODE]: Bottleneck at MX. N/8 foundations sufficient.

## 🔴 Must Do
- [strategy emoji ✅/⚠️/🔴/❓] [task] (🔓/⚙️/🚀 if hybrid) — [why this is must-do]

## 🟡 Should Do
- [strategy emoji] [task] — [why]

## 🟢 Could Do
- [strategy emoji] [task] — [why]

## ⏸️ Explicitly NOT Doing
- [deferred items from goal files — active deferral, visible]
```

### Priority Traffic Light
| Colour | Criteria |
|--------|----------|
| 🔴 Red | max(criticality, strategic_leverage) = HIGH |
| 🟡 Amber | Moderate criticality or leverage |
| 🟢 Green | Low criticality + low leverage, or easy wins |

Within each colour: worst cognitive match first (hardest while energy highest).

### Strategy Traffic Light (per task)
| Emoji | Meaning |
|-------|---------|
| ✅ | Aligned — strategy sound, just execute |
| ⚠️ | Suboptimal — strategy has gaps but proceeding anyway |
| 🔴 | Blocked — something upstream broken, may be wasted effort |
| ❓ | Undefined — strategy not mapped for this task |

To assign: trace task → path → mechanism step → sub-goal → goal. If trace is clean and goal is ACTIVE with no red foundations blocking this path → ✅. If foundations insufficient or bottleneck elsewhere → ⚠️ or 🔴.
```

### 2E. Weekly Review Triggers (NEW SECTION)

```markdown
## 📅 Weekly Strategy Review

Triggered weekly (day TBD by Jamie). This is the ONLY time strategy changes are made.

### Checklist:
1. Strategy sense-check: does current strategy still make sense?
2. Data pull: update mechanism chain metrics (throughput, conversion ratios)
3. Volume check: enough data to judge? (minimum thresholds from strategy-change-protocol.md)
4. Mechanism chain review: has bottleneck moved? Is intervention working?
5. Signals table: persevere vs change for each active path
6. Threshold gates: any approaching unlock? (≥80% = prepare, 100% = fire)
7. Deferred work: any unlocked items to activate?
8. Decision log: pending review dates this week?
9. Foundations: any maturity changes?

### Output:
- Updated goal files (metrics, bottleneck, foundation states)
- Decision log entries for any changes made
- Updated threshold gate statuses
- Any new tasks created from unlocked gates
```

## 4. Additions to SOUL.md

Add a new section:

```markdown
## System Discipline

**Stick to the plan.** The system exists so you don't have to rethink everything every day. Trust it. Execute in execution mode. Strategise in strategy mode. Don't blur the boundary.

**Don't over-map.** Not everything needs a goal, mechanism chain, and 47 subtasks. Match the weight of the system to the weight of the work. A 10-minute task gets headlines. A business-critical initiative gets full spec + goal linkage.

**Capture, don't act.** When you spot something strategic during execution — a better approach, a flawed assumption, a new opportunity — write it down in the Decision Log as an observation. Don't pivot. The weekly review exists for exactly this reason.

**Foundations before tactics.** If a foundation is missing (no offer, no contract, no revenue model), no amount of execution downstream will work. Flag infrastructure gaps immediately. They're priority unblocks.

**One bottleneck at a time.** The mechanism chain has one bottleneck. Work on that. Everything else is secondary. Resist the urge to "make progress on everything" — that's how nothing gets done.
```
