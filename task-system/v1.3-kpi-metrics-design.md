> **Status:** PROPOSED

> **Objective:** Embed funnel maths as first-class objects in the task system to enable automated bottleneck detection, downstream projection, and data-driven strategy reviews.

## 1. Architecture: The Single Source of Truth (SSOT)

Metrics will live in a single, versioned JSON file per goal. While the Goal Markdown file displays the data for humans, the JSON file is the machine-readable truth used for computation.

**Location:** `~/Master/goals/GOAL-XXX.metrics.json`

### Why JSON?
- **Machine Readable:** Crons can parse it without regex-hacking markdown tables.
- **Structured:** Supports nested mechanism chains and historical snapshots.
- **Dependency-Aware:** Allows defining "Computed" fields (e.g., Conversion Ratio = Output / Input).

## 3. Data Flow: Update Mechanism

Metrics are updated via a "Push/Pull" hybrid model:

### A. Automated Pull (Expandi/APIs)
- **Cron:** `SYNC-EXPANDI-H1` (🏢 Domain)
- **Action:** Fetches raw numbers from Expandi, updates the `input_value` and `output_value` for M1 in the JSON.
- **Script:** `scripts/metrics-update.py --source expandi --goal GOAL-001`

### B. Semi-Automated Pull (H1 Tracker)
- **Cron:** `DAILY-INTEGRITY` (🔧 System)
- **Action:** Reads the H1 Google Sheet (via existing tool or CSV export), maps columns to M2/M3 steps.
- **Script:** `scripts/metrics-update.py --source h1 --goal GOAL-001`

### C. Manual Push
- **Trigger:** Jamie updates a number in the Goal Markdown file.
- **Handler:** `DAILY-INTEGRITY` notices the MD table has changed, syncs the values back to the JSON SSOT.

## 4. Consumption: How Crons Use Metrics

### 1. Bottleneck Detection (Weekly Review Prep)
- **Logic:** For each step in the chain, check `ratio` against `target_ratio`. If `ratio < 80% of target` OR `throughput < 100% of requirement`, flag as BOTTLENECK.
- **Output:** Populates the "Current Bottleneck" field in the Goal MD.

### 2. Threshold Gate Checking (Morning Briefing)
- **Logic:** Read `metrics.json` → Check if M2 `output_value` >= Gate `condition`.
- **Output:** Unlocks deferred paths/tasks in the Morning Briefing.

### 3. Downstream Projection (Interactive / Strategy)
- **Logic:** If M1 input changes (35 -> 50), the projection engine multiplies the new input by the existing chain ratios:
  - `M1_Out * M2_Ratio * M3_Ratio = New_Goal_Output`
- **Output:** "Warning: At current ratios, increasing outreach to 50/day will result in 1.2 additional clients/week. Ensure M3 capacity exists."

## 6. Dependency Logic (Mathematical)

The system treats the mechanism chain as a **Vector of Ratios**.
- Let $R = [r_1, r_2, r_3]$ be the conversion ratios for steps M1, M2, M3.
- Let $I$ be the input at M1.
- Final Output $O = I \times r_1 \times r_2 \times r_3$.
- If any $r_n$ or $I$ changes, $O$ is recalculated instantly.
- **Inverted Logic:** If the user sets target $O$, the system calculates required $I = O / (r_1 \times r_2 \times r_3)$.
