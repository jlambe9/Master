# v1.3 Cron Registry — Automation Enforcement Map

> **Version:** 1.1
> **Date:** 2026-02-22
> **Author:** ATLAS v1
> **Purpose:** Complete map of every cron required to keep the v1.3 task system self-enforcing. Identifies what exists, what's missing, and what's not even mentioned in the system overview.

## Cron Classification

Every cron has a **Scope** field:
- **🔧 System** — Global task manager infrastructure. Exists because the task system exists. Without it, the system breaks regardless of domain.
- **🏢 Domain** — Serves a specific business function. Without it, that domain breaks but the task system keeps working. Created/deleted with domains.

**The test:** If you deleted all business tasks and goals, would this cron still need to exist? If yes → 🔧 System. If no → 🏢 Domain.

## Model Tiers

Every cron specifies a model tier:
- **Strategy (Opus 4.6)** — Never for crons. Interactive strategy work only.
- **Judgment (Sonnet)** — Crons needing context + decisions (briefings, reviews).
- **Validation (Flash)** — Checks, syncs, scans (near-free).

## 8. Key Finding: The Enforcement Gap

The v1.3 system overview describes a comprehensive, self-reinforcing task system. But **13 of 24 required crons don't exist yet**, including 3 critical ones. This means:

- **Threshold gates never fire automatically** — deferred work stays deferred forever unless Jamie manually checks
- **Completion integrity is unenforced** — tasks can be marked done without verified outputs and nothing catches it
- **No one watches the watchers** — if a cron silently fails, nothing detects it
- **Process maturity is manual** — the "cron-derived" maturity levels from the Atlas persona are aspirational, not real
- **Weekly review prep is ad-hoc** — the 10-point review checklist has no automated pre-computation

The existing 11 crons cover the **daily execution loop** well. The gaps are in **strategic enforcement**, **integrity verification**, and **meta-monitoring**.
