# Strategy Change Protocol

> **Version:** 1.0
> **Framework:** v1.3 Goal System
> **Purpose:** Every goal file links here. This is the reference for deciding whether to persevere or change.
> **Rule:** You cannot change strategy from execution mode. Capture doubts → act in strategy review.

---

## The Core Discipline

Strategy changes happen in **strategy mode only** (weekly/monthly reviews).
Execution mode is for executing, not deliberating.

If you have a strategic insight during execution:
1. Write it down (goal file Decision Log, or daily memory note)
2. Keep executing the current plan
3. Review the insight at the next scheduled strategy review
4. Only then decide whether to act on it

**Why:** Changing direction mid-execution is the #1 productivity killer for ADHD founders. You mistake the discomfort of doing hard work for a signal that the strategy is wrong. Usually it isn't — you just haven't done enough reps yet.

## Signals Table: Persevere vs Change

### Signals to PERSEVERE (execute harder)

| Signal | What It Means |
|--------|---------------|
| Leading indicators moving in the right direction | Strategy is working, just slowly |
| Haven't completed 2 full cycles yet | Insufficient data to judge — keep going |
| Blocker is execution, not strategy | You know WHAT to do, you're just not doing enough of it |
| Doubts are emotional, not data-driven | FEAR/PANIC system firing, not genuine strategic insight |
| You keep changing plans before completing one cycle | THIS is the problem — you're deliberating, not executing |
| Volume threshold not met yet | Can't judge conversion until minimum sample size reached |

### Signals to CHANGE THE PLAN

| Signal | What It Means |
|--------|---------------|
| Leading indicators flat/declining after 2+ cycles | Mechanism isn't converting despite execution |
| Surprising observations (expected input → unexpected output) | Your model of reality is wrong |
| External environment changed (market, competition, regulation) | Terrain shifted, old map is wrong |
| Consistent prospect/client feedback contradicts assumptions | Direct evidence value proposition is off |
| Bottleneck is structural, not effort-based | More effort won't fix a broken mechanism step |
| Cost of continuing > cost of pivoting | Sunk cost fallacy check |

## Strategy vs Execution Boundary

**How to tell which problem you have:**

| If the blocker is... | Then it's... | Action |
|---------------------|-------------|--------|
| A task status (CAPTURED / BLOCKED / not started) | **Execution problem** | Do the task. Unblock it. Assign it. |
| A conversion ratio (doing the work but not converting) | **Strategy problem** | Wait for strategy review. Analyse the mechanism step. |
| Missing foundation (no offer, no booking link) | **Infrastructure problem** | Priority unblock — this gates everything downstream |
| Emotional resistance (don't want to do it) | **Behavioural problem** | Use intervention stack (EBAM). Not a strategy issue. |

## Type 1 / Type 2 Decision Framework

*From Amazon's decision taxonomy.*

### Type 2 (Reversible) — Decide Fast, Change Fast
- Adding/dropping/pausing a **path**
- Changing task priority or approach
- Trying a new outreach method
- Adjusting a message template

**Protocol:** Make the decision. Log it. Review in 1-2 weeks. Reverse if wrong. Low cost.

### Type 1 (Irreversible) — Deliberate, Get Data First
- Changing the **goal** itself (different metric, different deadline)
- Changing the **mechanism chain** structure (adding/removing steps)
- Committing significant money or time (>1 week's effort)
- Pivoting the business model (coaching → consulting, 1:1 → group)

**Protocol:**
1. Write down: what specifically would you change?
2. What data supports this change? (Not feelings — data.)
3. What's the cost of being wrong?
4. Have you met the volume threshold to judge?
5. Sleep on it. Revisit at next strategy review.
6. If still compelling with data → change. Log decision with full rationale.

## Minimum Data Requirements

**Do not judge a strategy until you have enough data.**

| Decision | Minimum Volume | Why |
|----------|---------------|-----|
| "Is this outreach channel working?" | ≥50 contacts made | Small samples are noise |
| "Is our conversion rate real?" | ≥20 conversations | Statistical minimum for ratio |
| "Should we split a mechanism step?" | ≥20 data points at that step | Need to see where drop-off actually is |
| "Is this path better than that path?" | ≥30 contacts per path | Can't compare without comparable volume |
| "Is the offer wrong?" | ≥5 qualified prospects rejected | Distinguish offer problem from reach problem |
| "Should we add a new path?" | MVP path operational + ≥4 weeks data | Don't dilute before proving the mechanism |

**Rule:** If you haven't hit the minimum volume, the answer is "keep executing, not enough data yet." Full stop.

## The Change Protocol (When You've Decided to Change)

### Step 1: Document the Decision
In the goal file Decision Log:
- Date
- What's changing (be specific)
- What data drove this decision
- What you expect the change to produce
- Is this Type 1 or Type 2?
- Review date (when you'll judge if this was right)

### Step 2: Update the Goal File
- If changing a path: update Paths table, move old path to Path Archive with final metrics
- If changing mechanism chain: update the chain, note the old ratios for comparison
- If changing strategy summary: update Stick-to-Plan section

### Step 3: Cascade to Execution
- Pause/drop tasks on old path
- Create tasks for new approach
- Update task `Goal:` fields if paths changed

### Step 4: Set Review Date
- Type 2 changes: review in 1-2 weeks
- Type 1 changes: review in 4 weeks minimum
- Add review date to Decision Log AND OODA Cadence table

### Step 5: Commit
- Execute the new plan without second-guessing until the review date
- If new doubts arise before the review date → write them down, don't act
- At review date: check data against expectations from Step 1

## PDCA Integration

Every strategy change is a PDCA cycle:

1. **Plan:** Document what you're changing and what you expect (Steps 1-2 above)
2. **Do:** Execute the new plan (Step 3)
3. **Check:** At review date, compare actual results to predictions (Step 5)
4. **Act:** If it worked → standardise. If it didn't → another cycle with new hypothesis.

**The critical rule:** Check comes before Act. You don't just DO and then CHANGE. You DO, then CHECK (did it work?), THEN decide whether to ACT (standardise or adjust).

## Weekly Strategy Review Checklist

*Run weekly in strategy mode. This IS the Observe + Orient + Decide phases of OODA.*

- [ ] **Strategy sense-check:** Does the current strategy still make sense given what I know now?
- [ ] **Data pull:** Update mechanism chain metrics (throughput, conversion ratios)
- [ ] **Volume check:** Do I have enough data to judge? (Check minimum thresholds above)
- [ ] **Mechanism chain review:** Has the bottleneck moved? Is the current intervention working?
- [ ] **Signals table:** Run through persevere vs change signals for each active path
- [ ] **Threshold gates:** Any approaching unlock? (≥80% = start preparing)
- [ ] **Deferred work:** Any unlocked items to activate?
- [ ] **Decision log:** Any pending review dates this week?
- [ ] **Path archive:** Any learnings to surface from dropped paths?
- [ ] **Foundations:** Any maturity changes this week?

---

## Monthly Goal Review Checklist

*Run monthly. Full strategic review — the only time goal-level changes are permitted.*

- [ ] **Goal still valid?** Has the objective, metric, or deadline changed meaning?
- [ ] **Sub-goals balanced?** Are all sub-goals progressing, or is one starving another?
- [ ] **Mode transition?** Should BUILDING → OPERATIONAL (check System Stop Criteria)?
- [ ] **Environmental scan:** Has the world changed enough to invalidate the strategy?
- [ ] **Foundations reassessment:** Any foundation that was "sufficient" now insufficient?
- [ ] **Path portfolio:** Right number of paths? Too many (diluted) or too few (fragile)?
- [ ] **Decision log review:** Were past decisions right in hindsight?

---

## Quarterly Environmental Check

*Has the world changed enough to invalidate the strategy entirely?*

- [ ] Market conditions: new competitors, regulatory changes, economic shifts?
- [ ] Technology changes: new tools, platforms, AI capabilities that change the game?
- [ ] Personal circumstances: capacity, motivation, financial runway changed?
- [ ] Customer feedback patterns: are the problems you solve still the ones people have?

If any of these produce a "yes" → schedule a Type 1 strategy review. Don't panic-pivot.
